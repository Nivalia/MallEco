## 需要迁移的问题

### 1. 目录结构问题

部分模块缺少标准目录结构:

- 缺少 `dto/` 子目录
- 缺少 `interfaces/` 子目录
- 缺少 `constants/` 子目录

### 2. Entity问题

- `src/common/base.entity.ts` 使用 UUID，需要改为自增ID
- 字段命名使用 `createTime/updateTime`，需要统一为 `createdAt/updatedAt`
- 缺少软删除支持

### 3. DTO问题

- 缺少验证装饰器
- 缺少Swagger注解
- 缺少分页/排序DTO复用

### 4. 错误处理问题

- 直接使用 `throw new Error()`
- 未使用统一的 `ErrorCode`
- 缺少业务异常类

### 5. 日志问题

- 缺少统一的Logger context
- 日志格式不统一

## 迁移计划

### 第一批 (高优先级)

1. **用户模块** - 核心用户体系
2. **商品模块** - 核心商品体系
3. **订单模块** - 核心订单体系

9. **内容模块**

### 第四批 (辅助模块)

10. **文件模块**
11. **物流模块**
12. **通知模块**

## 实施建议

1. **每次只迁移一个模块**，完成后测试验证
2. **先从简单模块开始**，积累经验
3. **保持API兼容性**，不改变接口签名
4. **及时提交代码**，出现问题可快速回滚
