# 模块通信规范

## 通信方式选择

| 场景       | 推荐方式        | 说明           |
| ---------- | --------------- | -------------- |
| 同步调用   | 直接注入Service | 同一进程内     |
| 异步通知   | EventEmitter    | 模块间解耦     |
| 跨进程通信 | Message Queue   | RabbitMQ/Kafka |
| 分布式     | RPC/gRPC        | 微服务架构     |

## 事件命名规范

格式: `{模块名}.{事件名}`

```typescript
// 订单模块事件
'order.created';
'order.paid';
'order.shipped';
'order.completed';
'order.cancelled';
'order.refunded';

// 用户模块事件
'user.registered';
'user.login';
'user.logout';
'user.updated';
'user.deleted';

// 支付模块事件
'payment.succeeded';
'payment.failed';
'payment.refunded';

// 商品模块事件
'product.created';
'product.updated';
'product.stock.changed';
'product.price.changed';
```

## 事件定义示例

```typescript
// events/order.events.ts
import { EventEmitter2 } from '@nestjs/event-emitter';
import { Injectable } from '@nestjs/common';

export interface OrderCreatedEvent {
  orderId: string;
  userId: string;
  totalAmount: number;
  items: Array<{
    productId: string;
    quantity: number;
    price: number;
  }>;
}

export interface OrderPaidEvent {
  orderId: string;
  paymentId: string;
  amount: number;
}

@Injectable()
export class OrderEvents {
  constructor(private eventEmitter: EventEmitter2) {}

  emitOrderCreated(event: OrderCreatedEvent) {
    this.eventEmitter.emit('order.created', event);
  }

  emitOrderPaid(event: OrderPaidEvent) {
    this.eventEmitter.emit('order.paid', event);
  }
}
```

## 事件监听示例

```typescript
// listeners/order.listener.ts
import { OnEvent } from '@nestjs/event-emitter';

export class OrderListener {
  @OnEvent('order.created')
  handleOrderCreated(payload: OrderCreatedEvent) {
    console.log('Order created:', payload.orderId);
    // 发送通知
    // 更新统计数据
  }

  @OnEvent('order.paid')
  handleOrderPaid(payload: OrderPaidEvent) {
    console.log('Order paid:', payload.orderId);
    // 发送订单支付成功通知
    // 触发库存扣减
    // 触发分销佣金计算
  }
}
```

## 模块间Service调用

```typescript
// orders/orders.service.ts
@Injectable()
export class OrdersService {
  constructor(
    @InjectRepository(Order)
    private readonly orderRepository: Repository<Order>,
    private readonly goodsService: GoodsService,  // 直接注入
    private readonly paymentService: PaymentService,
    private readonly userService: UserService,
  ) {}

  async create(createOrderDto: CreateOrderDto) {
    // 1. 验证商品库存
    await this.goodsService.checkStock(createOrderDto.items);

    // 2. 计算价格
    const amount = await this.calculateAmount(createOrderDto);

    // 3. 检查用户余额（如果使用余额支付）
    if (createOrderDto.useBalance) {
      await this.userService.checkBalance(createOrderDto.userId, amount);
    }

    // 4. 创建订单
    const order = await this.orderRepository.save({...});

    // 5. 触发支付
    await this.paymentService.createPayment(order);

    return order;
  }
}
```

## 响应式调用模式

对于耗时操作，使用异步处理:

```typescript
// 使用返回 Promise
async findAll(query: ListQueryDto): Promise<PaginatedResponse<Order>> {
  const [list, total] = await this.orderRepository.findAndCount({
    where: query.toCondition(),
    order: query.toOrder(),
    skip: query.skip,
    take: query.take,
  });
  return PaginatedResponse.create(list, total, query.page, query.limit);
}
```

## 模块依赖原则

1. **单向依赖**: 上游模块不依赖下游模块
2. **依赖注入**: 使用依赖注入而非直接实例化
3. **接口隔离**: 通过接口定义依赖契约
4. **避免循环**: 严格禁止模块间循环依赖

```
依赖方向:
用户模块 ← 订单模块 ← 支付模块
                  ↓
             商品模块
```

## 错误处理

模块间调用时的错误处理:

```typescript
async createOrder(dto: CreateOrderDto) {
  try {
    const order = await this.orderRepository.save(dto);
    return order;
  } catch (error) {
    // 业务异常转换
    if (error instanceof InsufficientStockException) {
      throw new BusinessException(ErrorCode.GDS_STOCK_INSUFFICIENT, error.message);
    }
    throw error;
  }
}
```
