# 配置管理规范

## 配置分类

| 类别       | 命名空间     | 存放位置                      | 示例                             |
| ---------- | ------------ | ----------------------------- | -------------------------------- |
| 环境配置   | `env`        | `.env` 文件                   | `DB_HOST`, `REDIS_HOST`          |
| 应用配置   | `app`        | `config/app.config.ts`        | `app.name`, `app.version`        |
| 数据库配置 | `database`   | `config/database.config.ts`   | `database.host`, `database.pool` |
| 缓存配置   | `cache`      | `config/cache.config.ts`      | `cache.ttl`, `cache.prefix`      |
| 第三方配置 | `thirdparty` | `config/thirdparty.config.ts` | `wechat.appId`, `alipay.appId`   |

## 配置优先级

```
运行时环境变量 > .env文件 > 默认配置文件
```

## 配置使用示例

```typescript
import { ConfigService } from '@nestjs/config';

@Injectable()
export class UserService {
  constructor(private configService: ConfigService) {
    // 读取环境变量
    const dbHost = this.configService.get<string>('DB_HOST');

    // 读取带默认值的配置
    const dbPort = this.configService.get<number>('DB_PORT', 3306);

    // 读取嵌套配置
    const wechatAppId = this.configService.get<string>('wechat.appId');
  }
}
```


## 配置验证

使用 `class-validator` 验证配置:

```typescript
import { IsString, IsNumber, IsBoolean, validateSync } from 'class-validator';
import { plainToInstance } from 'class-transformer';

class AppConfig {
  @IsString()
  APP_NAME: string = 'MallEco';

  @IsNumber()
  APP_PORT: number = 3000;

  @IsBoolean()
  APP_DEBUG: boolean = false;
}

export function validate(config: Record<string, any>) {
  const validatedConfig = plainToInstance(AppConfig, config, { enableImplicitConversion: true });
  const errors = validateSync(validatedConfig, { skipMissingProperties: false });

  if (errors.length > 0) {
    throw new Error(errors.toString());
  }

  return validatedConfig;
}
```

## 敏感配置

敏感配置（如密码、密钥）必须:

1. 存储在环境变量中，不提交到代码仓库
2. 使用 `.env.example` 文件记录需要的变量名
3. 生产环境使用密钥管理服务

```bash
# .env.example
DB_HOST=localhost
DB_PASSWORD=  # 必须从环境变量读取
WECHAT_API_KEY=  # 必须从环境变量读取
```

## 环境特定配置

```typescript
// config/database.config.ts
import { registerAs } from '@nestjs/config';

export default registerAs('database', () => ({
  host: process.env.DB_HOST,
  port: parseInt(process.env.DB_PORT, 10) || 3306,
  username: process.env.DB_USERNAME,
  password: process.env.DB_PASSWORD,
  database: process.env.DB_NAME,

  // 生产环境额外配置
  ...(process.env.NODE_ENV === 'production' && {
    ssl: { rejectUnauthorized: true },
    extra: { connectionLimit: 20 },
  }),
}));
```
